## 引言
Scm.Net 是面向企业级应用开发的 .NET 10 快速开发框架，专注于中后台管理系统与多租户、多模块的企业应用。其设计理念围绕"可扩展、可插拔、可运维"的多层架构展开，通过统一的配置中心、动态 Web API、ORM 集成、任务调度、实时通信与插件体系，为企业提供从数据建模到业务编排的一体化能力。

该框架在企业应用开发中的定位是"高内聚、低耦合"的基础设施平台，既可作为独立后端服务，也可与前端生态（如 Vue3）配合构建一体化解决方案。相比传统框架，Scm.Net 的优势体现在：
- 统一的环境与配置模型，简化部署与运维
- 动态 Web API 与 DTO/VO 分层，降低重复开发成本
- 基于 SqlSugar 的 ORM 集成与单元化事务过滤器
- Quartz 任务调度与日志闭环
- SignalR 实时通信与 Hub 扩展
- 插件化扩展点（图像、音频、视频等）
- 完善的认证授权与跨域策略

**更新** 当前文档管理模式已调整，原有完整的项目概述文档系统已被删除，当前仅保留核心项目信息作为框架的基础介绍。

## 项目简介

### 框架背景与发展历程
Scm.Net 框架源于作者在电商供应链系统开发过程中的实践经验积累。作者先后开发过 OMS、WMS、TMS、DMS、BMS、RMA、ERP 等多款软件，对电商供应链有较深刻的理解。在此基础上，作者借鉴不同开源框架的优势，结合自身理解，重新定制并完善了 Scm.Net 框架，期望为后来的开发者提供帮助。

### 核心理念
- **企业级应用导向**：专注于中后台管理系统和多租户企业应用开发
- **模块化设计**：采用多层架构，各模块职责明确，便于扩展和维护
- **开发效率优先**：通过自动化工具和模板减少重复劳动
- **可扩展性**：支持插件化扩展和自定义功能开发
- **运维友好**：提供完善的日志、监控和配置管理机制

### 技术栈概览
- **运行时与框架**：.NET 10、ASP.NET Core
- **ORM**：SqlSugar
- **实时通信**：SignalR
- **任务调度**：Quartz.NET
- **日志**：Serilog
- **图像处理**：ImageSharp
- **JSON 序列化**：Newtonsoft.Json
- **映射**：Mapster
- **动态 API**：Scm.Server.Api
- **认证授权**：JWT Bearer
- **跨域**：CORS
- **缓存**：自定义缓存服务
- **插件**：图像/音频/视频插件

## 核心特性

### 动态 Web API
Scm.Server.Api 提供强大的动态 Web API 能力，能够自动发现服务并生成控制器：
- **自动控制器生成**：基于服务接口自动生成 API 控制器
- **约定式路由**：支持 RESTful 风格的路由约定
- **灵活的控制器选择**：可根据需求选择不同的控制器生成策略
- **配置化控制**：通过选项配置控制 API 生成的行为

### ORM 集成与数据访问
基于 SqlSugar 的 ORM 集成提供了完整的数据访问解决方案：
- **实体映射**：支持复杂对象关系映射
- **查询优化**：提供 LINQ 查询和原生 SQL 支持
- **事务管理**：内置事务支持和工作单元模式
- **数据库适配**：支持多种数据库类型

### 任务调度系统
Scm.Server.Quartz 提供企业级任务调度能力：
- **多种触发器**：支持一次性、重复、Cron 表达式等多种触发方式
- **作业持久化**：支持内存和数据库两种存储模式
- **集群支持**：支持 Quartz 集群部署
- **日志监控**：完整的作业执行日志和监控

### 实时通信
Scm.Server.SignalR 提供实时双向通信能力：
- **Hub 架构**：基于 Hub 的实时通信模式
- **连接管理**：支持连接池和连接状态管理
- **消息广播**：支持组播和点对点消息传输
- **扩展性**：支持水平扩展和负载均衡

### 文件管理系统
Nas.Server 提供企业级文件管理功能：
- **NAS 文件存储**：支持网络附加存储的文件管理
- **文件同步**：提供文件同步和版本控制功能
- **权限管理**：基于用户和角色的文件访问控制
- **搜索功能**：支持文件名、类型、内容的全文搜索

### 插件化扩展
Scm.Plugin.* 提供丰富的插件扩展能力：
- **图像处理**：支持图像格式转换、压缩、水印等
- **音频处理**：支持音频格式解析和元数据提取
- **视频处理**：支持视频格式转换和元数据获取
- **统一接口**：所有插件遵循统一的接口规范

## 应用场景

### 企业中后台管理系统
Scm.Net 最主要的应用场景是企业中后台管理系统，包括：
- **权限管理系统**：用户、角色、菜单、权限的统一管理
- **工作流引擎**：支持复杂的业务流程审批
- **数据字典管理**：统一的数据字典和配置管理
- **系统监控**：提供系统运行状态的监控和告警

### 多租户 SaaS 平台
框架天然支持多租户架构，适合构建 SaaS 平台：
- **租户隔离**：数据和配置的完全隔离
- **资源管理**：支持多租户的资源配额管理
- **计费系统**：集成化的计费和账单管理
- **多语言支持**：完整的国际化和本地化支持

### 内容管理系统
基于 Scm.Net 的内容管理能力：
- **文档管理**：支持各种格式文档的存储和管理
- **媒体库**：图片、音频、视频的统一管理
- **版本控制**：文档的版本管理和历史追踪
- **协作功能**：支持多人协作编辑

### 数据分析平台
框架提供的数据分析能力：
- **指标统计**：业务指标的实时统计和展示
- **报表系统**：自定义报表和数据导出
- **数据可视化**：丰富的图表和仪表板
- **预警系统**：基于规则的业务预警

### 移动应用后端
Scm.Net 可以作为移动应用的后端服务：
- **RESTful API**：标准的 RESTful 接口设计
- **认证授权**：支持多种认证方式
- **推送通知**：集成推送服务
- **文件上传**：支持大文件的断点续传

## 核心组件

## 性能考虑
- **ORM 与查询优化**：通过 SqlSugar 的实体键类型与枚举适配，减少数据库类型不匹配导致的隐式转换开销；利用 AOP OnLogExecuting 进行 SQL 观察与调优。
- **缓存策略**：通过缓存配置与服务扩展，结合业务场景进行热点数据缓存，降低数据库压力。
- **实时通信**：SignalR 在高并发下建议结合连接池与消息分区策略，避免单点瓶颈。
- **任务调度**：Quartz 作业应避免长时间阻塞操作，必要时拆分为轻量任务并异步执行。
- **文件与静态资源**：EnvConfig 对数据目录与 URI 的统一管理有助于静态资源映射与缓存命中率提升。
- **NAS 文件管理**：NAS 服务应合理配置文件索引和缓存策略，优化大文件的读写性能。

## 故障排查指南
- **启动失败**：检查 EnvConfig 的数据目录、日志目录、上传目录是否可写；确认数据库文件是否存在且可访问。
- **认证失败**：核对 JwtConfig 的 Issuer、Audience、Security 是否与前端一致；检查中间件顺序与令牌传递方式。
- **跨域问题**：核对 CorsConfig 的 AllowedOrigins/Methods/Headers 设置；确认预检缓存时间与凭据配置。
- **API 未生成**：检查 Scm.Server.Api 的动态 Web API 扩展是否正确注册，服务是否实现接口约定。
- **任务调度异常**：检查 QuartzConfig 的 BaseDir/DataDir/LogsDir 是否存在且可写，作业文件路径是否正确。
- **NAS 文件访问错误**：检查 NAS 数据库连接配置，确认文件路径映射和权限设置。
- **关于服务内容显示异常**：确认 about 目录下的文件是否存在，检查文件编码和内容格式。
